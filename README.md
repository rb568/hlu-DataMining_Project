# 데이터마이닝 7팀 주식 주가 예측 분류 모델 

본 저장소는 데이터마이닝 프로젝트의 분석 과정 및  
코드를 체계적으로 관리하기 위한 목적으로 만들어졌습니다.

## 1. 커밋 규칙
모든 커밋은 아래 규칙을 준수합니다

| 타입 | 의미 |
| :--- | :--- |
| **[Docs]** | 보고서 및 문서 수정 |
| **[Fix]** | 버그 수정 및 코드 개선 |
| **[Pre]** | 데이터 수집 및 전처리 관련 코드 |
| **[Data]** | 데이터셋 수집 및 원본 파일 관리 |
| **[Feat]** | 예측 모델링, 학습 알고리즘 및 기능 구현 |
---

## 2. 폴더 구조
본 프로젝트는 데이터 처리 및 
분석 흐름에 따라 다음과 같이 작성 되어있습니다.
| 파일 | 목적 |
| **[Models]** | 학습 및 모델링 코드 |
| **[Data]** | 원본 데이터 셋 및 전처리 완료 |
| **[Scripts]** | 자동화 도구 및 파이프라인 관리 |
| **[Analysis]** | 탐색 데이터 분석 및 시각화 코드 |
| **[Pre-processing]** | 데이터 수집 및 원본 파일 코드 |

## 3. 프로젝트 관리 규칙
1. **가독성** : 복잡한 로직은 '.[Scripts]' 폴더에 모듈화 하여 관리합니다.
2. **보안** : API키나 비밀번호 코드가 안에 있다면 반드시 삭제하고 올리세요.
3. **코드 기록** : 모든 '.ipynb'파일은 코드 위에 명확한 주석 및 설명을 포함합니다.
4. **보고서 연동** : 보고서 본문의 부록 언급과 깃허브 경로를 매칭하여 관리합니다.
